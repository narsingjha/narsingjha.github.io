This project was done for exploring the possibilities with shortwaves routine in CAM4 (radabs is the longwave routine in CAM4). Here, I ported the complete shortwave routine to the Xeon Phi, followed by the asynchronous execution of the routine. Although the frequency of execution of shortwaves routine is half that of the original simulation (for radabs, this was 1/12 of the original simulation), still I wanted to see the errors and performance improvement. Although the performance of complete CAM 4 was improved by ~10%, the errors were quite high.

For the synchronous execution of shortwave routine, major problem was posed by memory. I applied several optimizations like code reorganization for effective cache use, removing useless memory copies, loop tiling, loop fission and interchanging array dimensions for efficient vectorization, approximating exponent functions and dead code removal, which gave an improvement of 5% over the original shortwaves executions. More details of the project will be updated soon.
