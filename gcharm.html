G-Charm is a generic framework, based on Charm++. It's runtime system contains the following optimizations -
a. Dynamic Scheduling of the work on CPU and GPU
b. Memory management on the GPU
c. Reducing the data transfer across CPU and GPU by maximum data reuse on the GPU
d. Combining multiple kernel invocations into a single one to avoid communication between heterogeneous resources and to utilize the GPU more efficiently.

G-Charm also provides a programming interface, wherein the programmer writes separate GPU CUDA kernel and CPU Charm++ functions. G-Charm has been demonstrated on three highly useful scientific applications -
a. 2D Jacobi Solver (Regular application)
b. Dense Matrix Cholesky Factorization (Regular application)
c. Molecular Dynamics Simulation (Irregular application)

My work focuses on improving the efficiency of the current state of G-Charm, a research prototype. It also aims at adding more application support to G-Charm. Ultimately, it has to be converted into a fully-fledged open source software. Find the related paper here.
